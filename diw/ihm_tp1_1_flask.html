utilisation de flask

utiliser pycharm (à l’IUT /opt/pycharm/bin/pycharm.sh et licence server)


créer une nouvelle application flask (attention : l’explorateur de fichiers de pycharm a parfois du mal à scanner le contenu du dossier HOME monté en NFS)

utiliser python3 comme interpréter

cette application se présente sous la forme suivante

  • le dossier static contient tous les éléments (contenus) dit “static” du site : images, fichiers CSS, fichiers javascript utilisé par les pages HTML ….)
  • le dossier template contient les pages HTML, ces pages HTML sont appelées gabarit ou template
  • attention : pas de dossier venv




  • Ouvrir un terminal dans pycharm et tester l’application avec la commande ci-dessous
python app.py

première application

  • modifier le fichier app.py avec comme contenu :
from flask import Flask, request, render_template, redirect, url_for, abort  # application WSGI

# (interface de serveur web python)
# comportements et méthodes d'un serveur web


app = Flask(__name__)    # instance de classe Flask (en paramètre le nom du module)

@app.route('/')
@app.route('/page1')
def show_page1():
     return render_template('tp1/tp1_page1.html')

if __name__ == '__main__':
    app.run(debug=True, port=5000)
  • créer un dossier tp1 dans le dossier static
  • créer un dossier tp1 dans le dossier templates


  • recopier le tp1(fichiers HTML) dans le dossier templates


  • tester l’application : ouvrir un terminal syr pycharm et lancer la commande ci-dessous
python app.py
  • cliquer sur le lien dans le terminal

lancer le serveur web ; la boite à outils

  • tester les commandes ci-dessous :
hostname -i
ip addr
  • tester l’url sur la machine de votre voisin


Créer un fichier launcher.sh avec les droits d’exécution : recopier le contenu ci-dessous dans un terminal de pycharm

touch launcher.sh
chmod u+x launcher.sh

recopier le contenu ci-dessous dans le fichier launcher.sh : voir (documentation)

flask --debug  --app app  run   --host 0.0.0.0
  • lancer ce fichier pour tester l’application (utiliser le bouton dans l’editeur)

fichiers “static”


ça fonctionne mais les contenus “static” n’apparaissent pas


Les fichiers STATIC sont tous les fichiers dont le contenu ne changent jamais : les fichiers CSS (vos styles, bootstrap), les images, les fichiers javascript


  • déplacer les fichiers “static” : images, fichiers CSS et javascript dans le dossier tp1/static

il faut modifier les fichiers html qui sont des gabarits (templates)

<link rel="icon"  href="{{ url_for('static', filename='tp1/iconeweb.ico') }}" />

<a href="/page2"><img src="{{ url_for('static', filename='tp1/linux.jpg') }}" alt="mon image devrait apparaitre" title="sympa linux!" width="250" height="200" /></a>

Un gabarit (template) est un fichier dont le contenu va être modifié par l’application, l’application va passer des paramètres (données) à la vue : la vue est renvoyée au client

présentation NodeJS

https://nodejs.org/en/

  • NodeJs, c’est un interpréteur de JavaScript hors du navigateur. Il pourrait être utilisé pour faire des applications comme : conversion image, backup, accès fichiers …).
  • NodeJs est surtout utilisé pour faire du serveur (web)
    • Full JS (coté client et coté serveur)
    • librairie pour faire du serveur
  • NodeJs utilise le moteur V8 de Google chrome pour interpréter le JavaScript (plus rapide que d’autres interpréteurs)
  • NodeJs est très Événementiel
objet.emit('myEvent',a,b);   // émettre un evt
objet.on('myEvent', function(a,b){} );  // fonction de callback
  • NodeJs peut être Asynchrone. Les serveurs web PHP( django, ruby on rails …) sont synchrones.

  • NodeJS est autosuffisant en serveur web (PHP utilise un serveur web comme Apache plus mature)

  • NodeJs est fondé sur une communauté qui utilise les NPM (express, jade, socket.io, mocha)

  • les versions de nodeJS changent rapidement

  • pour plus d’informations

Découvrir la programmation Asynchrone avec NodeJs

  • Remplacer l’instruction readdirSync par readdir. Ce script génère une erreur car readdir est asynchrone : on ne peut pas afficher le résultat avant de l’avoir lu.

  • Remplacer

var fs = require('fs');
files = fs.readdir('.');
for (var i=0;i<files.length;i++){
  console.log(files[i]);
}
  • Par le code ci dessous qui utilise une callback
var fs = require('fs'); 

fs.readdir('.', function(err, files) {  // fonction en asynchrone
  if(err) {
    console.log('Un probleme est arrive');  // tester avec un mauvais dossier
  }
  else{        // console.log(files);
    for (var i=0;i<files.length;i++){
      console.log(files[i]);
    }
  }
});

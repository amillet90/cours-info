Il faut corriger le nom de la variable : remplacer m par min

  • Dans le fichier de l’application que l’on lance (app.py), il n’y a que des fonctions sauf app.run qui lance l’application
  • Les annotations permettent de définir des routes
  • La fonction qui se trouve dans le code juste en dessous une ou plusieurs routes est associée à cette(ces) route(s) : la fonction est lancée si la route est trouvée
  • Si plusieurs route porte le même nom, c’est la première trouvée par le programme qui est utilisée (pas d’erreurs)
  • Si aucune route est trouvée, l’application affiche une erreur


  • le mot clé return devant render_template est obligatoire
  • sur jinja, le moteur de template :
    • les variables sont interprétées
    • si une variable n’existe pas, rien n’est affiché

fichiers “static”


ça fonctionne mais les contenus “static” n’apparaissent pas


Les fichiers STATIC sont tous les fichiers dont le contenu ne changent jamais : les fichiers CSS (vos styles, bootstrap), les images, les fichiers javascript


  • déplacer les fichiers “static” : images, fichiers CSS et javascript dans le dossier tp1/static

il faut modifier les fichiers html qui sont des gabarits (templates)

<link rel="icon"  href="{{ url_for('static', filename='tp1/iconeweb.ico') }}" />

<a href="/page2"><img src="{{ url_for('static', filename='tp1/linux.jpg') }}" alt="mon image devrait apparaitre" title="sympa linux!" width="250" height="200" /></a>

Un gabarit (template) est un fichier dont le contenu va être modifié par l’application, l’application va passer des paramètres (données) à la vue : la vue est renvoyée au client
